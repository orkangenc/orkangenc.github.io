Changing listener port for an Oracle database

This exampel sets the port to 3000


$ srvctl stop listener
$ srvctl modify listener -l LISTENER -p "TCP:3000"
$ srvctl config listener
Name: LISTENER
Network: 1, Owner: oracle
Home: <CRS home>
End points: TCP:3000

$ srvctl stop scan_listener
$ srvctl modify scan_listener -p 3000
$ srvctl modify listener -l LISTENER_SCAN1 -p "TCP:3000"
$ srvctl start listener
$ srvctl start scan_listener

Do the same for all instances


SQL> alter system set local_listener='(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=172.30.33.51)(PORT=3000))))' scope=both sid = 'instance1';
SQL> alter system set remote_listener='instance-scan.company.com:3000' scope=both;

SQL> alter system set local_listener='(DESCRIPTION=(ADDRESS_LIST=(ADDRESS=(PROTOCOL=TCP)(HOST=172.30.33.52)(PORT=3000))))' scope=both sid = 'instance2';
SQL> alter system set remote_listener='instance-scan.company.com:3000' scope=both;

$ srvctl stop database -d instance 
$srvctl start database -d instance

Update the new port in tnsnames.ora for all instances
